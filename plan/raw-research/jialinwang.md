- test cases

    - should contain both normal and edge cases, e.g. a player is playing a slot machine normally vs someone is intentionally breaking the game
    - exact test steps
    - expected results, e.g. tokens increase by 60, player loses 60 hp, etc

- offer light / dark mode
- include a game design document so it is easier for the users to test
